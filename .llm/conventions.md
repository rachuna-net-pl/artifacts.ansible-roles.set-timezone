# Conventions: set-timezone

## Variable naming

| Prefix | Location | Purpose |
|--------|----------|---------|
| `in_` | `defaults/main.yml` or passed at play level | Input parameters ‚Äî values provided by the caller to configure role behaviour |
| `var_` | `vars/main.yml` | Internal role variables ‚Äî computed or fixed values not intended to be overridden |

### Examples

```yaml
# defaults/main.yml ‚Äî caller-facing inputs
in_timezone: Europe/Warsaw
in_locale: pl_PL.UTF-8
in_language: pl_PL:pl
in_lc_all: pl_PL.UTF-8

# vars/main.yml ‚Äî internal role variables (not overridable)
var_locale_gen_path: /etc/locale.gen
```

### Rules

- Never use unprefixed variable names inside the role ‚Äî all variables must carry `in_` or `var_` prefix to avoid collisions with other roles and host variables.
- `in_` variables must always have a default defined in `defaults/main.yml`.
- `var_` variables must be defined in `vars/main.yml` only ‚Äî never in `defaults/main.yml`.
- Do not pass `var_` variables from outside the role.

### Key rules

- `tasks/main.yml` is the sole entry point for task logic. Split into separate files (e.g. `tasks/locale.yml`, `tasks/timezone.yml`) only when the file exceeds ~80 lines and logical grouping is clear.
- `handlers/main.yml` must contain all handlers. Reference them via `notify:` in tasks.
- `meta/main.yml` must declare `platforms`, `min_ansible_version`, and `dependencies` (empty list `[]` if none).
- No `library/`, `lookup_plugins/`, or `filter_plugins/` directories unless the role genuinely requires custom plugins.
- No `templates/` or `files/` directories unless the role requires static files or Jinja2 templates.

## Task conventions

- Use FQCN (Fully Qualified Collection Name) for all modules, e.g. `ansible.builtin.apt`, `ansible.builtin.copy`, `ansible.builtin.command`.
- Guard platform-specific tasks with `when: ansible_facts.os_family == "Debian"` (or appropriate fact).
- Use `ansible.builtin.stat` + conditional instead of shell `test` commands.
- `changed_when: false` is allowed only for truly read-only commands. For commands that modify state, use `changed_when` with an explicit condition or register output to detect changes.
- Prefer `ansible.builtin.copy` with inline `content:` over `ansible.builtin.template` for simple single-value files.
